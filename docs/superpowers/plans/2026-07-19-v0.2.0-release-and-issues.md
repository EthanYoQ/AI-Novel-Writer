# v0.2.0 Release and Issue Resolution Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Integrate the startup and bilingual work, publish a verified `v0.2.0` Windows release, then reply to and close Issues #2, #3, and #4.

**Architecture:** The release commit is created only after source, packaged-native, executable-smoke, and README gates pass. GitHub mutations occur in order: push commit, create tag/release, verify the live asset, then comment and close each issue.

**Tech Stack:** Git, GitHub CLI/API, pnpm, Electron Builder, PowerShell, SHA-256.

## Global Constraints

- Target tag is exactly `v0.2.0`; package version is exactly `0.2.0`.
- Asset name is exactly `AI-Novel-Writer-0.2.0-windows-x64.zip`.
- Do not claim resolution or close an Issue before the live release asset is independently verified.
- Push only intended tracked project changes; keep `.agents/`, `AGENTS.md`, `.superpowers/`, and local archives out of commits.
- If a release gate fails, stop the release sequence and leave all Issues open.

### Task 1: Bump and verify version metadata

**Files:**
- Modify: `package.json`
- Modify: `pnpm-lock.yaml`
- Modify: `README.md`
- Modify: `README_zh.md`

**Interfaces:**

- [ ] **Step 1: Write the failing version contract**

Create `src/shared/__tests__/release-version.test.ts`:

```ts
import { readFileSync } from 'node:fs'
import { describe, expect, it } from 'vitest'

describe('v0.2.0 release metadata', () => {
  it('keeps package, lock, and readme asset names aligned', () => {
    const pkg = JSON.parse(readFileSync('package.json', 'utf8'))
    const lock = readFileSync('pnpm-lock.yaml', 'utf8')
    const readmes = readFileSync('README.md', 'utf8') + readFileSync('README_zh.md', 'utf8')
    expect(pkg.version).toBe('0.2.0')
    expect(lock).toMatch(/version:\s+0\.2\.0/)
    expect(readmes).toContain('AI-Novel-Writer-0.2.0-windows-x64.zip')
  })
})
```

- [ ] **Step 2: Run and verify RED**

Run: `pnpm exec vitest run src/shared/__tests__/release-version.test.ts`

Expected: FAIL because package version is `0.1.0`.

- [ ] **Step 3: Update version and lock**

Run: `pnpm version 0.2.0 --no-git-tag-version`

Confirm About and status bar consume `__APP_VERSION__` rather than hard-coded versions. Update README release links if required.

- [ ] **Step 4: Run the contract and commit**

Run: `pnpm exec vitest run src/shared/__tests__/release-version.test.ts`

Expected: PASS.

